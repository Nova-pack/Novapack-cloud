PASO 1 — CRITICO: Activar Autenticacion Anonima

Sin este paso, la app de reparto NO mostrara albaranes cuando se entre con Master PIN.

  1. Abrir la consola de Firebase:
    https://console.firebase.google.com/project/novapack-68f05/authentication/providers
  2. En la lista de proveedores, buscar "Anonymous" (Anonimo)
  3. Hacer click en el icono de editar (lapiz)
  4. Activar el toggle "Enable"
  5. Pulsar "Save" (Guardar)

Por que: El Master PIN no usa SMS ni email, necesita una sesion Firebase minima para que Firestore permita leer los albaranes. La auth anonima crea esa sesion sin necesidad de datos del usuario.

PASO 2 — ALTA: Configurar Sentry (Monitorizacion de Errores)

Sentry captura automaticamente cualquier error JS que ocurra en la app y lo registra en un dashboard.

  1. Crear cuenta gratuita en https://sentry.io (5.000 errores/mes gratis)
  2. Crear un proyecto nuevo: Platform → JavaScript (Browser)
  3. Copiar el DSN que te proporcionan. Formato:
    https://xxxxx@oXXXXXX.ingest.sentry.io/XXXXXXX
  4. Editar el archivo public/sentry-config.js y pegar el DSN:
    window.NOVAPACK_SENTRY_DSN = "TU-DSN-AQUI";
  5. Desplegar: npx firebase deploy --only hosting --project novapack-68f05

Resultado: Cada error en admin, cliente o repartidor aparecera en el dashboard de Sentry con la pagina, linea exacta, navegador del usuario y stack trace completo.

PASO 3 — MEDIA: UptimeRobot (Alerta si la Web se Cae)
  1. Crear cuenta gratuita en https://uptimerobot.com (50 monitores gratis)
  2. Click en "Add New Monitor"
  3. Configurar:
    - Monitor Type: HTTP(s)
    - Friendly Name: Novapack Cloud
    - URL: https://novapaack.com
    - Monitoring Interval: 5 minutes
  4. En Alert Contacts, anadir tu email
  5. Pulsar "Create Monitor"

Resultado: Recibiras un email inmediato si la web se cae y otro cuando vuelva a funcionar. Tambien genera informes mensuales de uptime.

PASO 4 — MEDIA: Configurar Alertas en Sentry

Una vez completado el Paso 2:

  1. En Sentry, ir a Settings → Alerts
  2. Crear nueva regla: "When new issue is created → Send email"
  3. Opcional: filtrar por nivel error o fatal para evitar spam
  4. Puedes crear reglas separadas para cada modulo (admin, client, driver) usando el tag novapack_module
PASO 5 — BAJA: Firebase App Check (Anti-Abuso)
  1. Ir a Firebase Console → App Check
  2. Registrar la app web con reCAPTCHA v3
  3. Activar enforcement para Firestore y Storage

Resultado: Bloquea peticiones desde clientes no autorizados (bots, scripts maliciosos). Solo la app real puede acceder a los datos.

PASO 6 — BAJA: Backups Automaticos de Firestore
  1. Firebase Console → Firestore → Backups
  2. Para backup manual desde terminal:
    gcloud firestore export gs://novapack-68f05.appspot.com/backups/FECHA
  3. Para backups automaticos: requiere plan Blaze + Cloud Scheduler
  4. Recomendacion: backup semanal minimo, diario si hay volumen alto
